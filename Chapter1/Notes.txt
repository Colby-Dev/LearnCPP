1.1) Statement and the structure of a program

--> Introduction <--

	In this chapter, we cover the basics to every C++ program. Because the topics are very numerous and hard to cover all them these are split up to optimize the topic. Later topics will be explored in more detail as they are covered. 

--> Statements <--


	A compute program is a sequence of instructions that tell the computer what to do. A statement is a type of instruction that causes the program to perform specific actions.Statments are the most common type of instruction in a C++ program. This is because they are the smallest independent unit of computation in C++. Most, but not all, statements in C++ end in a semicolon. C++ is compiled into many machine language instructions from statements. 

	For advanced readers: 

		- Declaration statements
		- Jump statements 
		- Expression statements
		- Compound statements
		- Selection statements (conditionals)
		- Iteration statements (loops)
		- Try blocks

--> Functions and the main function <--

	In C++, statements are typically grouped into units called functions. A function is a collection of statements that get executed sequentially (in order, from top to bottom). As you learn to write your own programs, you'll be able to create your own functions and mix and match statements in any way you please (we'll show how a future lession).

When the program is run, the statements inside of main are executred in sequential order. Programs usually terminate after the last statement inside main has been executed, even though some may abort early. Functions will typically be to perform specific parts of a task.

--> Characters and text <--

	Early computers were just text calculations. In modern times they are designed for much more complex task such as typing written languages. These are called strings. 

--> Dissecting Hello World! <--

	#include <iostream>

	int main() { 

		std::cout << "Hello World!!";
		return 0;
	}

	Line 1 shows a special line called the preprocessor directive. This #includes preprocessor directive indicates that we want the standard iostream library. We need this for the cout to print to console. 

	Line 2 is blank, and is ignored by the compiler. This line exists only to help huamns read the code.

	Line 3 tells the complier that we're going to write (define) a function whose name (identifier) is main. As stated before the program must have a main function so the complier has an entry. This function will produce a number which is why we initialize the function with int and return 0; 

	Lines 4 and 7 denote the curly braces showing what will be contained inside of the main function. 

	Line 5 is the first statment in the main function. std::cout or standard::character out, outputs information to the console. In this case it will be Hello World!!

	Line 6 is a return statement. This sends an alert to the OS that the function is exiting. The value 0 is the standard sucess value which lets the OS know that the function executed successfully. 

--> Syntax and syntax errors <--

	Rules that programs follow are designed around certain spaces and symbols. This is called syntax. The C++ programming language syntax describes how elements must be written and arranged in order for the program to be considered valid and run. Otherwise this is considered a syntax error. 

1.2) Comments
-------------------------------------------------------------------------------------------

	A comment is a programmer readable note that is used to inform readers of blocks of code. This note is not recognized by the complier so syntax constraints do not apply. 

> Single line comments

	Single line comments only apply to individual lines in the progam. These are used for short notes or commenting out peices of code. Single lined comments are written by using the // syntax.

--> Multi line comments <--

	Multi-line comments are used for commenting many lines without using the // operator for each line. Often lines are numerous so using a multi-lined comment is beneficial for reading programs or putting extensive notes. The operator used to do this is the /* */. Everything inside of the two stars will be commented. 

1.3) Introduction to objects and variables
-------------------------------------------------------------------------------------------

--> Data and values <--

	In 1.1, we discussed that programs are just instructions and statements, and that functions are just collections of statements that handle specific parts of the functions. Programs actually produce results by manipulating data, data being any information that can be moved, written, and stored. 

	A program can acquire data to work with in many ways, such as a file, database, network etc. A single peice of this data is called a value or data value. Common examples are numbers, single characters ('') , or text (""). 

	Values placed directly into the code are called literals. Literals are covered in detail in 5.2. Literals are the easiest way to provide raw values for your program, but they have shortcomings. Literals are read-only values, so their values can't be modified. 

--> Random Access Memory <--

	The main memory in a computer is called Random Access Memory (RAM). When programs run the operating system loads the program into RAM. Any data that is hardcoded into the program is loaded at this point. 

	The OS also allocate some additional RAM for the program to use while it runs. Common uses for this memory are to store values entered by the user, to store data read in from a file or network, or to store values calculated while the program is running so they can be used again later. RAM can be thought of as a series of numbered boxes that can be used to store data while the program is running. 

--> Objects and Variables <--

	In C++, direct memory access is discouraged. We access memory indirectly using objects. An object represents an area of storage that can hold a value. This allows us to interact with only the object, and lets the compiler sort out how to retrieve the value.

	Objects can be unnamed (anonymous), but more often we name our objects that we create using an identifier. An object with a name is called a variable. 

--> Variable Definition <--

	A definition statement can be used to tell the compiler that we want to use a variable in our program. This is called defining a variable.

	At compile time (when the program is compiled), the compiler gets the data type and name of the variable we gave. The compiler itself then handles all the other details like the storage of the object, how much memeory, creation / deletion, etc. 


--> Variable Creation <--

	At runtime (when the program is loade into memory and run), each object is given a storage location. This process is called allocation. Once this happens the object is created and can be used. 

--> Data types <-- 

	In C++, the type of an object must be known at compile time, so that the compiler knows what the data type of the object is and how much memory it requires. INT data types hold whole numbers. 

1.4) Variable Assignement and Initialization
-------------------------------------------------------------------------------------------

--> Variable assignment: 

	After defining a variable, you can give it a value. The variable will hold the value that it is given and can be referenced later. This is called assignment and is done with one = sign. By default assignment operator = is a copy assignment as it copies the value on the right hand side of the operator. 

--> Variable Initialization 

	The downside of assigning a variable in its own step is that it produces more redundant code. These steps can be combined. This is called initialization. By using the data type, variable name, two curly braces {}, and finally the data the variable holds. 

	int width {5};

--> Different Forms of Initialization

	Initialization in C++ is complex. There are 5 common forms of initialization: 

		1) default initialization - int a

		2) copy initialization - int b = 5

		3) direct list initialization - int c {5}

		4) direct initialization - int d (6)

		5) value initialization - int e {};

--> Default Initialization

	When no initializer is provided. This leaves the variable with no determinate value or garbage value. 

--> Copy Initialization

	Using the assignment operator (=) this copies the value from the rigt hand side to the left hand side. This is less efficient as creating the copy requires the compiler to create a register in memory for the copy when it is being initalized. 

--> Direct Initialization

	Using paraenthesis to assign variables with data. This has fallen out of favor for direct list initalizations, due to its broader use case. Direct initalization is for values explicitly cast to another type. 

--> List Initialization

	The most modern way of list initialization. This works well as it clearly states that we are initalizing an object that will hold data, thus whenever there are {} we know we're initalizing something. 

	This also is helpful because the complier will catch any errors or warnings from the progam at compile time. One of these would be using int a {4.5} which should be a double but user would be warned at compile time. 

--> Value Initialization and Zero Initialization

	When initialized values are set to an empy data value such as {}, the compiler will initalize the object to default values, which are usually 0 (zero initialization) but can be non zero if the default values are defined like in classes. 

--> Instantiation

	Basically meaning a variable is created, allocated, and initalized. This is considered an instance of the object named as said variable name. This can be applied any other class type objects but can broadly stand for any object. 

--> Initializing Multiple Variables

	Initalization of multiple variables at once can happen. Doing so requires the variable to be initalized with the value for each specific variable. Example of this: 

		int a {5}, b{5} // Both are initalized to be 5

--> Unused Initialized Variables Warnings

	Modern compilers will throw warnings if variables are initialized but now used. Some settings will treat these warnings as errors. If this error is being shown, consider removing the variable, or understanding its purpose. 

--> The [[maybe_unused]] Attribute

	In the event we are using variables that are going to be used later in the program, we can use the [[maybe_unused]] attribute before the initialization of the variable to avoid the compiler complaining about the use. The biggest benefit is that the compiler will optimize these variables out to improve performance. 

		double pi {3.14} //Used var
		[[maybe_unused]] string velocity {dx/dy} // Unused var

1.5) Introduction to iostream: cout, cin, and endl
-------------------------------------------------------------------------------------------

--> The Input/Output Library

	The I/O library is part of the C++ standard. This library allows for user inputs and outputs. This is included in the header file #iostream. The normal program has these headers put first in most programs. 

--> Std::cout

	The cout predefined variable is used to send data to the console to display. cout stands for character output. C++ uses insertion operators << to take in the value that will be outputted. 

--> Using std::endl to Output a Newline

	The endl predefined variable is used to break lines on the output of the console. This behavior will cause the cursor to move to the next line. 

--> Std:cout is Buffered

	The output of text is buffered in a queue to the console. The output is stored in a region of memory set aside to collect request. The buffer periodically flushes the data collected, which forces all the data to the console. Endl flushes the buffer automatically. 

	If the program crashes the buffer is flushed but no data is displayed. Bufferring in this way is optimal as it allows data to flow efficiently not all at once. 

--> Std::endl vs \n 

	Using endl is inefficient as it performs 2 jobs, moving the cursor to a new line and flushing the buffer. Using multiple endl at the end of statements will continue to do these 2 jobs which is not optimal. 

	Using \n will not flush the buffer and is therefore ideal when outputting many objects to the console. \n can be added to the end of strings to break line as the compiler reads this as a single linefeed chat (LF) 

--> Std::cin

	The cin predefined variable is used to read input from the keyboard. We typically use the extraction operator >> to put the input data in a variable. Defining and initalizing a variable to hold user input is needed to use user data. 

		int x{};
		std::cin >> x; 

		cout << x << endl;

	cin is used for basic input as end users must press the enter key in order to submit data to the program and cin. Third party libraries are used to gain other functionality. 

--> std::cin is buffered

	Inputting data also has a two staged process similar to cout. The inputted characters are added to the end of the buffer. The enter key press is also passed the \n character to break the line. 

	The >> operator removes characters from the front of the buffer and assigns them via copy assigned to the associated variable.
